# Viewflow Cookbook Map

Use the Viewflow cookbook as an example source, not as code to copy blindly.

## Simple Workflow Examples

- Look for minimal start, task, and completion flows to understand basic node structure.
- Adapt naming, permissions, and business rules to the local project.

## Complex Workflow Examples

- Use complex examples to reason about branching, joins, approvals, and rework loops.
- Verify that examples match the installed Viewflow version and local project patterns.

## Custom UI Components

- Review examples for form/view/template integration patterns.
- Keep business rules testable outside UI-only code.

## CRUD/Viewset Examples

- Use CRUD and viewset examples to understand URL, view, and dashboard behavior.
- Confirm permissions and task actions are enforced server-side.

## Testing Examples

- Prefer examples that show process starts, transitions, permissions, and final states.
- Add project-specific tests rather than relying on cookbook behavior.

## Notes

- Do not copy large cookbook code into Karakana skills.
- Link to or summarize examples in project documentation when needed.
